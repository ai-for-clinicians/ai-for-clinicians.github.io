A vision for the future

How the clinical knowledge graph could get started. Populating and maintaining the graph. Governance and ensuring future viability. Not having to anticipate all possible use cases, but instead allowing extensibility. Getting buy-in from essential stakeholders. Allowing a gradual phase-in, rather than a wholesale change.

What the system would need

Publication layer and annotation layer. A process for ingesting new publications as they become available, using as much machine-readable metadata as possible. A light-weight, but sufficiently feature-rich annotation ontology to offer core functionalities. Processes for loading info from the graph and writing to it. An API that allows building applications that we envision, e.g. search, automatic aggregation and enrichment by machines.

What the system could do

A platform, not a product

Separate application layer from data layer to allow different agents to build applications with different functionalities. Don't try to anticipate and prescribe what they want to do with the graph.

Governance and viability

Compare the principles for open scholarly infrastructures to get some input about what could be necessary in this area.
